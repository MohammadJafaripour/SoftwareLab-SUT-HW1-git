1. Project Overview & Objectives

The core objective of this laboratory assignment is to build a robust mental model of Git architecture, collaborative workflows, and Continuous Integration/Continuous Deployment (CI/CD) pipelines. Instead of relying on abstraction layers provided by modern IDE graphical interfaces, this project was developed entirely inside the Command Line Interface (CLI).


By executing foundational "plumbing" and "porcelain" Git commands manually, we gained absolute visibility over structural transitions across the three local Git states (Working Directory, Staging Area, Local Repository) and the upstream Remote Server. The practical artifact chosen for this lab is a static HTML/CSS developer dashboard tracking our specific assignment metrics.

2. Core Requirements & Engineering Decisions

3. Step-by-Step Terminal Execution Log

Step 1: Initializing the Project Workspace

Initialize a clean local directory and set up basic repo tracking rules.

git init software-lab-1 cd software-lab-1 echo -e "node_modules/\n.DS_Store\ndist/" > .gitignore git add .gitignore git commit -m "chore: initialize project workspace and .gitignore"

Step 2: Setting up Upstream Cloud Remotes

Link the local repository with the remote cloud source repository.

git remote add origin https://github.com/mohammad-jafaripour/software-lab-1.git git remote -v

Step 3: Establishing the Semantic Branch Architecture

Isolate development tasks from the highly stable production track.

git checkout -b dev git checkout -b feature/dashboard-ui

Step 4: Iterating Atomic Commit Logs (Example Sequence)

Saving progressive changes step-by-step to build history milestones.

git add index.html git commit -m "feat: design dashboard grid framework layout" git add style.css git commit -m "style: apply dark theme color palette variables"

Step 5: Handling and Resolving Merge Conflicts

Manually auditing converging code blocks when data conflicts occur.

git checkout feature/dashboard-ui git merge dev # Terminal returns: "CONFLICT (content): Merge conflict in index.html" # Open index.html, clean the separation block markers manually, then: git add index.html git commit -m "fix: resolve text grid overlap merge conflict manually"

Step 6: Upstream Sync and Pull Request Processing

Push safe features upstream to invoke structured code inspection paths.

git push origin feature/dashboard-ui git checkout dev git pull origin dev

4. CI/CD Automated Deployment Configuration

To achieve automated pipeline operations, the following automated script environment configuration was declared under .github/workflows/deploy.yml to power real-time updates directly into the GitHub Pages hosting service upon code approval.

name: Deploy Static Site to GitHub Pages on: push: branches: - main permissions: contents: read pages: write id-token: write concurrency: group: "pages" cancel-in-progress: true jobs: deploy: environment: name: github-pages url: ${{ steps.deployment.outputs.page_url }} runs-on: ubuntu-latest steps: - name: Checkout Source Tree uses: actions/checkout@v4 - name: Setup Cloud Pages Node uses: actions/configure-pages@v4 - name: Upload Static Web Assets uses: actions/upload-pages-artifact@v3 with: path: '.' - name: Deploy Live Instance to Production id: deployment uses: actions/deploy-pages@v4

CI/CD Actions Deployment Workflow

  • Shared Repository Integration All registered team members concurrently collaborate on code changes using a centralized GitHub remote hosting platform destination.
  • Peer Review Validation Isolate features into staging branches, push updates upstream, and propose integration changes exclusively via Pull Requests.
  • Automated Pipeline Trigger GitHub Actions picks up accepted merges to the main branch automatically via configured workflow parameters.
  • Static Cloud Hosting The automation runner compiles assets and updates the running application interface live on GitHub Pages.

5. Git Command Reference

Search or filter by category to find the Git command you need.

Setup & Init

git init [name] Create a new local repository in a new folder
git clone [url] Clone a remote repository locally
git config --global user.name Set your global author name for commits
git config --global user.email Set your global author email for commits

Staging & Committing

git status Show working tree status and staged changes
git add [file] Stage a specific file for the next commit
git add . Stage all changed files in the working directory
git commit -m "[msg]" Record staged snapshot with a descriptive message
git commit --amend Modify the most recent commit message or content

Branching & Merging

git branch List all local branches
git branch -a List all local and remote branches
git checkout -b [name] Create and switch to a new branch
git merge [branch] Merge a branch into the current active branch
git branch -d [name] Delete a fully merged local branch

Remote Operations

git remote add origin [url] Link local repo to a remote server
git push origin [branch] Upload local branch commits to the remote
git pull origin [branch] Fetch and merge remote branch into current branch
git fetch Download remote changes without merging

History & Inspection

git log --oneline --graph Compact visual graph of commit history
git diff Show unstaged changes in the working directory
git diff --staged Show changes staged but not yet committed
git show [hash] Display metadata and diff for a specific commit
